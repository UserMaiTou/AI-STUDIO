# RESKIN_STARTER_PACKET v0

## What This Is

This is a minimal starter packet for a real reskin project validation run.

It exists to test whether the AI_STUDIO Core + Domain idea works in practice.

This packet exists to validate Core + Domain assumptions.

Its purpose is learning and evidence collection.

It is not a finalized operating model.

It is not a complete reskin operating system.

It is not `DOMAIN_RESKIN`.

It does not authorize reskin work by itself.

---

## When To Use

Use this packet only when Human has approved a real reskin project trial.

The packet should help the project team observe:

- which AI_STUDIO Core rules help
- which Core rules do not fit reskin work
- what future Domain guidance may be needed
- what reusable Skills may be worth creating later

---

## Required Core Reads

Before using this starter packet, read:

1. `core/PROJECT_HANDOFF.md`
2. `core/TOOL_ROUTING.md`
3. `core/SESSION_PROTOCOL.md`
4. `AGENTS.md`
5. relevant mission or task files, when assigned

---

## Authority

Human Final Decision remains the highest authority.

Repository files remain the source of truth.

Tool is capability, not authority.

Executor and Reviewer are separate responsibilities.

No Agent may treat this packet as permission to perform account, signing, legal, privacy, advertising, or store-submission decisions.

---

## Not DOMAIN_RESKIN

This packet is intentionally small.

It should collect evidence from a real reskin project before AI_STUDIO decides whether a formal `DOMAIN_RESKIN` layer is needed.

Do not expand this packet into a full Domain system without Human approval and a dedicated TASK.
