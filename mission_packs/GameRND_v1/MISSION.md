# GameRND_v1 Mission

> This Mission Pack is intended to support a real new game R&D project.
> The project team executes the project.
> AI maintains project evidence and retrospective inputs.
> Protected decisions remain with Human Director.

---

## 1. Mission Objective

Support a new game R&D project from initial intake through first playable validation, iteration, technical validation, project decision, and retrospective.

The objective is to create evidence for product judgment, not just activity.

During execution, AI should continuously collect evidence about:

- whether AI-STUDIO Core guidance is useful for new game R&D
- whether the product hypothesis is testable
- whether the core gameplay has enough validated promise
- what R&D-specific guidance is missing
- what should remain project-specific
- what may later become Domain guidance
- what may later become reusable Skills
- what objective validation can be handled before Human Review

This mission does not create `DOMAIN_GAMERND`.

This mission does not finalize an operating model.

---

## 2. Expected Deliverables

The project team should return the full Mission Pack package when it is used as part of an exported `AI_MISSION/` folder.

Expected project outputs include:

- project intake summary
- product hypothesis
- core gameplay definition
- prototype scope
- first playable build or playable equivalent
- Human feel probe notes
- iteration notes
- technical validation notes
- project decision recommendation
- AI-maintained project observations
- AI-generated retrospective inputs and completion package
- known risks and unresolved issues

If a deliverable is not applicable, mark it as `N/A` and explain why.

---

## 3. Required Observation Collection

During execution, AI should maintain project observations and record:

- what was tried
- what changed
- what felt promising
- what felt weak
- what objective evidence was checked
- where the team got blocked
- what required Human escalation
- what should be reused or avoided next time

Use `GAME_RND_OBSERVATION_LOG_TEMPLATE.md` only as supporting structure when useful.

AI must record observations continuously during project execution instead of reconstructing them from memory at the end.

---

## 4. Required Retrospective Inputs

After the project completes, AI generates project retrospective inputs and completion-package material.

The retrospective inputs should summarize:

- final project outcome
- useful workflow steps
- unclear or wasteful workflow steps
- objective validation evidence that was collected
- subjective Human Review moments that shaped the project
- reusable patterns
- risks discovered
- recommended improvements for future GameRND Mission Packs

---

## 5. Human Escalation Requirements

The project team must escalate to Human Director before decisions involving:

- new project direction change
- core gameplay reversal
- business model change
- large new system expansion
- major technical direction change
- formal project approval
- project pause
- project cancellation
- account decisions
- SDK decisions
- privacy decisions
- compliance decisions
- key monetization decisions
- commit, push, release, or publishing authorization

No Agent may make final decisions on these topics.

Use `GAME_RND_ESCALATION_RULES.md` when unsure.
