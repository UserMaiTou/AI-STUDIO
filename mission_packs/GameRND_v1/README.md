# GameRND_v1

## What This Is

GameRND_v1 is a Mission Pack source for one real new game R&D validation run.

It helps a project team move from initial project intake through first playable validation, iteration, technical validation, project decision, and retrospective evidence collection.

It is not a project runtime, not a specific game project, and not a formal Domain.

It is not AI-STUDIO Core.

It does not finalize a game R&D operating model.

It does not authorize project approval, pause, cancellation, account, SDK, privacy, compliance, monetization, commit, push, release, or publishing decisions.

## When To Use

Use this Mission Pack only when Human Director has approved a real new game R&D trial.

The pack should help the project team and AI observe:

- whether AI-STUDIO Core guidance helps new game R&D work
- what R&D-specific guidance is missing
- what should remain project-specific
- what may later become Domain guidance
- what may later become reusable Skills
- which objective validation checks can reduce unnecessary Human Review load

## Authority

Human Final Decision remains the highest authority.

Repository or project-local files remain the source of truth for their own workspace.

Tools provide capability, not authority.

Executor and Reviewer are separate responsibilities.

AI should handle objective validation with available files, diffs, scripts, builds, logs, checklists, and recorded evidence whenever practical before asking Human Director for judgment.

Human Director should be asked for subjective judgment, strategic direction, protected decisions, final authorization, and risk acceptance.

## Not DOMAIN_GAMERND

This Mission Pack is intentionally small.

It should collect evidence from real new game R&D work before AI-STUDIO decides whether formal `DOMAIN_GAMERND` guidance is needed.

Do not expand this Mission Pack into a full Domain system without Human Review and a dedicated approved task.
