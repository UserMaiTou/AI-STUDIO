# GAME_RND_WORKFLOW_DRAFT

> Purpose: high-level workflow outline only.
> This file does not define Core session protocol or a formal Domain standard.

---

## Workflow

### 1. Project Intake

Capture the initial project idea, target audience, platform assumptions, constraints, success expectations, and known risks.

### 2. Product Hypothesis

State what product opportunity or player experience the project is testing.

Keep the hypothesis short enough to judge after a prototype cycle.

### 3. Core Gameplay Definition

Define the core action, player goal, feedback loop, and expected moment-to-moment experience.

Avoid expanding into a full content plan before the core experience is validated.

### 4. Prototype Scope

Define the smallest playable scope that can test the product hypothesis and core gameplay.

Clarify what is intentionally excluded.

### 5. First Playable

Build or assemble the first playable version or playable equivalent.

The goal is feel validation, not production completeness.

### 6. Human Feel Probe

Collect Human Director feedback on feel, clarity, friction, desire to continue, and biggest concern.

AI records observations continuously without forcing premature conclusions.

### 7. Iteration

Run focused iterations based on the strongest evidence from the feel probe and project goals.

Avoid broad expansion unless Human Director approves it.

### 8. Technical Validation

Check whether the current direction has manageable technical risk.

Document major risks, unknowns, dependencies, validation results, and system expansion needs.

### 9. Project Decision

Prepare a recommendation:

- continue
- iterate more
- pause
- cancel
- escalate for formal approval

Human Director makes the final decision.

### 10. Retrospective

AI generates project retrospective inputs and completion-package material.

Focus on what AI-STUDIO should learn for future projects.

---

## Validation Guardrails

This draft is not a checklist.

This draft is not a release gate.

AI should collect objective evidence whenever practical before asking Human Director for subjective judgment or protected decisions.

Do not ask Human Director to manually catch objective issues that AI can reasonably check through files, diffs, builds, logs, scripts, or checklists.

This draft does not authorize project approval, pause, cancellation, account, SDK, privacy, compliance, monetization, commit, push, release, or publishing decisions.
